| 服务器 | 专长 | 应用场景 |
|--------|------|----------|
| **Magic** | 现代UI组件生成 | 从21st.dev模式创建生产级、可访问的UI组件 |
| **Playwright** | 浏览器自动化和测试 | 端到端测试、浏览器兼容性、用户交互验证 |
| **Context7** | 文档和最佳实践 | 框架文档、编码标准、实施模式 |

## 💡 使用示例

### 示例1: 现代React应用开发
```
用户: "构建一个现代React电商应用，使用TypeScript，优化性能和可访问性"

工作流程:
1. 需求分析和技术选型: React + TypeScript + Vite + Tailwind CSS
2. 架构设计: 组件化架构、状态管理、路由设计
3. UI组件开发: 使用Magic MCP生成可访问组件和设计系统
4. 性能优化: 代码分割、图片优化、缓存策略
5. 测试策略: Jest + React Testing Library + Playwright
6. 部署监控: 性能监控、错误跟踪、持续优化

输出: 生产级React应用，优化的性能、全面的测试、监控设置
```

用户: "开发一个企业管理系统的前端，需要复杂的数据展示和权限管理"

1. 需求分析: 复杂数据表格、权限控制、实时更新需求
2. 架构设计: 微前端架构、状态管理、组件库设计
3. UI实现: 数据表格组件、权限控制界面、仪表板
4. 性能优化: 虚拟滚动、数据缓存、按需加载
